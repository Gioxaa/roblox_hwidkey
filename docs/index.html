HWID Licenser API

RESTful service for issuing, verifying, and revoking RS256 JWT licenses bound to hardware identifiers. Deploy on Node.js 18+ with npm, store revocation info on the filesystem, guard admin access using HTTP Basic + bcrypt, and optionally manage licenses from Discord.

Getting Started

  1. Generate RSA keys:
mkdir -p keys data/logs
openssl genrsa -out keys/private.pem 2048
openssl rsa -in keys/private.pem -pubout -out keys/public.pem
  1. Create your environment file:
cp .env.example .env

Update .env with admin credentials, JWT metadata, rate limiting, trust proxy mode, and key paths. Install dependencies and start:

npm install
npm start

Default listener: http://localhost:4000. Override with PORT.

Authentication

Admin endpoints require HTTP Basic Auth using ADMIN_USER and ADMIN_PASS. Plaintext passwords from the environment are hashed with bcrypt on boot and never persisted.

Endpoints

Public GET /healthz

Health probe; returns { "status": "ok" }.

Admin POST /issue

Issue a new license token bound to a hardware ID.

FieldTypeNotes
hwidstringRequired. ≤256 characters.
ttlSecondsnumberRequired. 60–5,184,000 seconds.
planstringOptional plan label.
notestringOptional operator note.

Response:

{
  "token": "",
  "jti": "uuid",
  "exp": 1700000000
}

Logs append to data/logs/issued.jsonl with timestamp, IP, JTI, HWID, plan, exp, and note.

Example:

curl -u "admin:changeme" \
  -H "Content-Type: application/json" \
  -d '{"hwid":"DESKTOP-ABC123","ttlSeconds":3600,"plan":"pro","note":"trial"}' \
  http://localhost:4000/issue
Public POST /verify

Validate a JWT license for the supplied hardware ID.

Request:

{ "token": "", "hwid": "DESKTOP-ABC123" }

Responses:

{ "ok": true, "plan": "pro", "exp": 1700000000 }
{ "ok": false, "reason": "invalid_or_expired" }
{ "ok": false, "reason": "hwid_mismatch" }
{ "ok": false, "reason": "revoked" }

Checks: RS256 signature, iss, aud, sub === "license", exp, HWID equality, and revocation list.

Admin POST /revoke

Revoke a license by JTI.

{ "jti": "c1345bf0-..." }

Persists the ID to data/revoked.json (atomic write) and logs data/logs/revoked.jsonl.

Public GET /status/:jti

Returns { "revoked": true | false } for a given license ID.

Offline Verification

Share keys/public.pem with clients for local token validation. Use the bundled helper:

node verify-offline.js "" "DESKTOP-ABC123"

The script enforces signature, iss/aud, subject, and HWID match.

Discord Bot (Optional)

Enable the bot by setting INCLUDE_DISCORD_BOT=true in .env and providing:

  • DISCORD_TOKEN and DISCORD_CLIENT_ID from the Developer Portal.
  • DISCORD_GUILD_ID for instant command registration (optional, otherwise commands register globally).
  • ISSUER_BASE_URL pointing to this API (for example http://localhost:4000).
  • ISSUER_BASIC_USER and ISSUER_BASIC_PASS used for Basic Auth when calling /issue and /revoke.

Launch with npm run bot. Slash commands open interactive modals:

  • /issue — collect HWID, TTL, plan, note. Replies ephemerally with metadata and a button to reveal the full token (available for five minutes).
  • /revoke — modal for the JTI, then revokes the license and clears cache.
  • /status — modal for the JTI, returns whether the license is active or revoked.

Operational Notes

  • Terminate TLS at a reverse proxy and align the server's TRUST_PROXY setting with your proxy hop count or CIDR.
  • Rotate RSA keys periodically; update JWT_KID to signal consumers.
  • Protect keys/ and data/ with restrictive filesystem permissions.
  • Ship JSONL audit logs to monitoring or SIEM tooling for traceability.
